|------|------|

| Flash | 4MB，无 PSRAM |
| MicroPython | 1.12.0 |
| 构建系统 | Makefile |

## OLED

### 几何与坐标

- 板载 OLED：1.3 英寸，SSD1106，`128x64` 单色像素，I2C 地址 `0x3C`
- 坐标原点在左上角：`x=0..127`，`y=0..63`
- 缓冲区：`oled.buffer`（bytearray，1024 bytes，`framebuf.MONO_VLSB`，8 页 x 128 字节/页）
- `oled.show()` 才把 FrameBuffer 送到屏幕；绘图和写字 API 默认只改内存缓冲区
- 像素颜色：`1` 点亮，`0` 熄灭；`oled.invert(1)` 是显示控制器级的全屏反相

### 文本尺寸

- `oled.text(s, x, y, c)` 是 MicroPython `FrameBuffer.text()`：只适合 ASCII，固定 `8x8` 像素，不支持改字体；128x64 最多按 16 列 x 8 行排布。
- `oled.DispChar()` 使用固件字库 `Noto_Sans_CJK_SC_Light16.bin`（Flash `0x400000`），字体高度 `16px`，名义宽度 `16px`，实际每个字符不等宽。
- `DispChar` 的常用行 y 坐标是 `0, 16, 32, 48`，也就是最多 4 行；不要用 `y=64`。
- 实测固件字库常见字符宽度：中文/中文标点如 `你`、`好`、`中`、`。`、`，` 通常字形宽 `11px`，`DispChar()` 每字后加 `1px` 间距，所以连续排版推进约 `12px`；`A` 推进 `7px`，`i` 推进 `4px`，`W` 推进 `10px`，`0` 推进 `7px`，空格推进 `3px`。
- 做网格布局时可保守按 `16x16` 单元排 8 列 x 4 行；做紧凑布局时用 `DispChar()` 返回值继续排，不要手算固定宽。

```python
oled.fill(0)
oled.DispChar("你好", 0, 0)   # 支持中文
oled.show()

# DispChar 参数
# mode: TextMode.normal / TextMode.rev / TextMode.trans / TextMode.xor
oled.DispChar(s, x, y, mode=TextMode.normal, auto_return=False)

oled.pixel(x, y, c)
oled.line(x1, y1, x2, y2, c)
oled.rect(x, y, w, h, c)
oled.fill_rect(x, y, w, h, c)
oled.text(s, x, y, c)      # 内置 8x8 ASCII 字体
oled.bitmap(x, y, bitmap, w, h, c)
oled.scroll(dx, dy)
oled.invert(v)
oled.contrast(c)            # 0~255
```

`DispChar()` 的第 4 个位置参数就是 `mode`，固件里 `TextMode` 是普通整数常量，很多旧代码会直接传数字：

| 写法 | 等价常量 | 效果 |
|------|----------|------|
| `1` | `TextMode.normal` | 前景点亮，背景清空 |
| `2` | `TextMode.rev` | 前景熄灭，背景点亮，常用于选中态 |
| `3` | `TextMode.trans` | 只画前景，不改背景 |
| `4` | `TextMode.xor` | 前景像素与当前缓冲区异或，背景不改 |

因此 `oled.DispChar("你好", 0, 0, 2)` 等价于 `mode=TextMode.rev`。第 5 个位置参数是 `auto_return`：

```python
oled.DispChar("你好", 0, 0, 2, True)  # rev + 自动换行
```

读旧代码时可能看到 `oled.DispChar("你好", 0, 0, 2, 1)`，其意图通常是 `mode=TextMode.rev, auto_return=True`。但源码使用 `auto_return is True` 判断，不是普通 truthy 判断；生成新代码时应写 `True`，不要依赖 `1` 被当作布尔值。

`DispChar()` 返回 `(字符总像素宽度, (next_x, next_y))`，适合连续排版：

```python
oled.fill(0)
w, pos = oled.DispChar("你好", 0, 0)
oled.DispChar("世界", pos[0] + 2, pos[1])
oled.show()
print(w, pos)
```

自动换行：

```python
oled.fill(0)
oled.DispChar("长文本自动换行显示", 0, 0, auto_return=True)
oled.show()
```

`auto_return=True` 时，当前字符放不下会换到 `x=0, y+=16`；超过底部后源码会回到屏幕顶部。做滚动列表时应自己分页，避免自动覆盖第一行。

### 反相显示

`oled.invert(v)` 是 SSD1106 硬件/显示控制器层面的全屏反相。源码向控制器发送 `SET_NORM_INV | (v & 1)`，即 `v=0` 正常显示，`v=1` 反相显示。

特性：

- 全屏生效，不能只反相某个区域或某个字。
- 不修改 `oled.buffer`，只是显示输出阶段反转；`oled.fill(0)` 后如果 `invert(1)` 仍开启，屏幕会显示为亮底。
- 通常调用后立即生效，不需要 `oled.show()`；状态会保持到 `oled.invert(0)`、重新初始化或复位。
- 适合整屏闪烁、提示、夜间反色等低成本效果。

局部反色用软件/FrameBuffer 层面的文字模式：

```python
oled.DispChar("选中", 0, 0, mode=TextMode.rev)
oled.show()

from font import digiface_30
oled.DispChar_font(digiface_30, "123", 0, 24, invert=True)
oled.show()
```

- `TextMode.rev` 会把 `DispChar()` 字符盒内的前景/背景写成反色，适合菜单选中态。
- `DispChar_font(..., invert=True)` 会先反转自定义字体 glyph 数据再写入缓冲区。
- 这些软件反相会改变缓冲区内容，必须 `oled.show()` 才显示；后续刷新仍保留这些像素，除非重新绘制或清屏。

### 自定义字体

```python
from font import digiface_30

oled.DispChar_font(digiface_30, "123", 0, 0)
oled.show()
```

- 函数签名是 `oled.DispChar_font(font, s, x, y, invert=False)`，不是把 `font` 放最后。
- 字体需由 `font_to_py.py -x` 转成水平映射（hmap）的 `.py` 字体模块；返回值同样是 `(字符总像素宽度, (next_x, next_y))`。
- 预置 `font/` 目录包含若干数码/点阵字体模块，如 `digiface_11/16/21/30/44`、`digiface_it_30/42`、`dvsm_12/16/21`、`dvsmb_12/16/21`。

### 性能优化库（可选）

⚠️ **默认使用标准 API**。只有在用户明确要求极致性能优化时才考虑以下库。

使用前必须询问用户：**"是否接受引入外部第三方库？这会增加依赖、RAM 占用和编译内存占用。"**

#### SSD1106 DPR（差分页刷新）

**来源：** https://github.com/emofalling/micropython-ssd1106_dpr

**原理：** 双缓冲区 + 页级差异检测，只传输变化的页到显示控制器

**性能收益：**
- 差分页刷新，只传输变化的页
- 降低功耗
- 提升刷新流畅度

**代价：**
- +1KB RAM（双缓冲区）
- +编译内存占用（.py 约占用更多 RAM）
- 外部依赖（需手动维护）

**适用场景：**
- 静态内容多（如菜单、仪表盘）
- 频繁局部更新（如数值变化、进度条）
- 不适合全屏动画

**安装：**
```bash
# 编译为 .mpy 减少内存占用
mpy-cross-v5 -march=xtensawin ssd1106_dpr.py
# 将 ssd1106_dpr.mpy 放入板子的 /lib 目录
```

**使用示例：**
```python
from ssd1106_dpr import SSD1106_I2C
import machine

i2c = machine.I2C(scl=machine.Pin(22), sda=machine.Pin(23))
oled = SSD1106_I2C(128, 64, i2c)

oled.text("Static", 0, 0)
oled.text("Content", 0, 10)
oled.show()  # 首次全屏刷新

# 后续只更新变化部分
oled.fill_rect(0, 20, 128, 10, 0)  # 清除旧数值
oled.text(str(value), 0, 20)
oled.show()  # 自动差分刷新，只传输变化的页
```

#### devlib（优化版 mpython.py）

**来源：** https://github.com/emofalling/devlib

**特性：**
- native 编译加速（更快的 OLED 刷新、文本渲染）
- I2C 超频支持（最高 1250KHz）
- 优化的文本渲染（`DispChar` 性能提升）

**代价：**
- 替换标准 `mpython.py`（可能影响兼容性）
- +编译内存占用
- 外部依赖

**适用场景：**
- 大量文本显示（特别是中文）
- 频繁 OLED 刷新
- 需要更高 I2C 速度

**安装：**
```bash
# 编译为 .mpy
mpy-cross-v5 -march=xtensawin devlib.py
# 将 devlib.mpy 放入板子的 /lib 目录
```

**使用示例：**
```python
from devlib import *  # 替代 from mpython import *

# 使用方式与标准 API 相同，但性能更高
oled.fill(0)
oled.DispChar("你好", 0, 0)
oled.show()
```

**性能优化决策树：**

```
遇到性能问题？
├─ 是 → 先尝试代码优化（见 patterns.md）
│   ├─ 仍不够 → 询问用户是否接受外部库
│   │   ├─ 是 → 根据场景选择 DPR 或 devlib
│   │   └─ 否 → 继续优化代码或降低需求
│   └─ 已解决 → 完成
└─ 否 → 使用标准 API
```
